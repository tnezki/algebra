Unit 8 Practice by Depth of Knowledge

DOK 1 — Rungs 1–2Students can recognize function families, vocabulary, notation, parent-function shapes, transformations, and basic key features.

Format: 3–5 minute warm-up, vocabulary check, representation recognition, model-type sort, or exit ticket

Purpose: Check whether students can recognize function families, transformations, absolute value features, model vocabulary, and basic representations.

Assessment Ideas

  1. Model Type Recognition Check Can Be Assessed After Section 1 – Students identify whether relationships appear linear, quadratic, exponential, absolute value, or inappropriate for a listed model.
  2. Transformation Vocabulary Check Can Be Assessed After Section 2 – Students identify shifts, reflections, stretches, compressions, and parent functions.
  3. Absolute Value Feature Check Can Be Assessed After Section 3 – Students identify vertex, symmetry, distance interpretation, and basic graph features.
  4. Function Family Review Check Can Be Assessed After Section 5 – Students sort graphs, equations, and tables by function family.
What this tells the teacher:
Students who struggle here need additional support with the language and visual features of function families before model selection and synthesis tasks.
1.

Use the graph. Match each label to a function family: linear, quadratic, exponential, or absolute value.

DOK 2 — Rungs 3–4Students can compare relationships, identify model types and transformations, and connect graphs, tables, equations, and contexts.

Format: Mini-quiz, graph/table task, matching task, partner check, or short written task

Purpose: Check whether students can compare function families, identify transformations, work with absolute value functions, and connect data to model types.

Assessment Ideas

  1. Choose a Model from Data Task Can Be Assessed After Section 1 – Students select a likely model type from a table or scatterplot and support the choice briefly.
  2. Transformation Match Task Can Be Assessed After Section 2 – Students match transformed graphs, equations, and descriptions.
  3. Absolute Value Graph and Context Task Can Be Assessed After Section 3 – Students graph or interpret an absolute value relationship in context.
  4. Mixed Function Comparison Task Can Be Assessed After Section 5 – Students compare functions using graph, table, equation, and context features.
What this tells the teacher:
Students at this level can generally organize representations and apply known tools, but the assessment reveals whether they connect those tools to model behavior.

DOK 3 — Rungs 5–6Students can justify model choices, analyze transformations and absolute value structure, critique models, and explain limitations using evidence.

Format: Constructed response, model critique, strategy comparison, representation analysis, or discourse prompt

Purpose: Check whether students can justify model choices, analyze transformations, interpret absolute value structure, and critique conclusions using evidence.

Assessment Ideas

  1. Model Justification Task Can Be Assessed After Section 1 – Students defend why a model type is appropriate or inappropriate using multiple representations.
  2. Transformation Explanation Task Can Be Assessed After Section 2 – Students explain how a transformation changes features across function families.
  3. Absolute Value Reasoning Task Can Be Assessed After Section 3 – Students justify solutions or intervals using distance and two-sided reasoning.
  4. Model Limitation Critique Can Be Assessed After Section 4 – Students critique a prediction or model conclusion and revise the reasoning.
What this tells the teacher:
This level reveals whether students can move beyond recognizing function families into defending decisions, interpreting features, and critiquing models.

DOK 4 — Rungs 7–8Students can synthesize multiple function families to choose, design, critique, revise, and defend mathematical models for unfamiliar situations.

Format: Brief performance task, modeling task, critique-and-revise task, design task, or capstone synthesis task

Purpose: Check whether students can synthesize Algebra 1 function knowledge to model unfamiliar situations, compare families, communicate limitations, and justify decisions.

Assessment Ideas

  1. Model Selection Challenge Can Be Assessed After Section 1 – Students choose an appropriate model for data and justify the choice using mathematical evidence.
  2. Transform and Compare Task Can Be Assessed After Section 2 – Students design transformed functions from different families and compare key features.
  3. Modeling Investigation Snapshot Can Be Assessed After Section 4 – Students analyze a small data set, select a model, make a prediction, and state limitations.
  4. Function Synthesis Capstone Task Can Be Assessed After Section 5 – Students analyze a mixed-representation situation and defend conclusions across function families.
What this tells the teacher:
DOK 4 tasks reveal whether students can synthesize the course, model with evidence, critique assumptions, and communicate decisions. These should remain short performance tasks rather than large projects.
1.

Create a data set that would be better modeled by a quadratic function than by a linear function. Include a table and explain the evidence.

Workspace
2.

Workspace
3.

Workspace
4.

Workspace
5.

Workspace

Workspace
8.
