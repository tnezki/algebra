Unit 1 Practice by Depth of Knowledge

DOK 1 — Rungs 1–2Students can recognize representations, vocabulary, function notation, inputs, outputs, and basic function-family shapes.

Format: 3–5 minute warm-up, vocabulary check, representation match, graph/equation identification, or exit ticket

Purpose: Check whether students can recall vocabulary, recognize representations, identify inputs and outputs, classify basic function-family shapes, and use notation correctly.

Assessment Ideas

  1. Representation Vocabulary Check Can Be Assessed After Section 1 – Students identify tables, graphs, equations, contexts, variables, and representation features.
  2. Input-Output Fluency Check Can Be Assessed After Section 2 – Students identify inputs and outputs and evaluate simple function rules.
  3. Function Family Recognition Check Can Be Assessed After Section 3 – Students recognize common function families from basic graphs and equations.
  4. Expression and Equation Notation Check Can Be Assessed After Section 5 – Students identify expression parts, equation structure, variables, and solutions.
What this tells the teacher:
Students who struggle here need additional support with the language and notation of functions, representations, expressions, and equations before moving into deeper modeling and justification.

DOK 2 — Rungs 3–4Students can evaluate function rules, match representations, classify function families, and write or rewrite expressions and equations.

Format: Exit ticket, partner check, whiteboard response, mini-quiz, matching task, or short written task

Purpose: Check whether students can evaluate function rules, match representations, rewrite expressions, classify function families, and write equations from situations.

Assessment Ideas

  1. Multiple Representation Match Task Can Be Assessed After Section 1 – Students match a table, graph, equation, and context that represent the same relationship.
  2. Function Rule Evaluation Task Can Be Assessed After Section 2 – Students evaluate a function rule for several inputs and interpret the outputs.
  3. Function Family Sorting Task Can Be Assessed After Section 3 – Students sort graphs and equations into function families and explain simple features.
  4. Equation Model Task Can Be Assessed After Section 5 – Students write and solve an equation from a context and interpret the solution.

DOK 3 — Rungs 5–6Students can justify whether representations match, analyze function behavior, explain expression structure, and critique models.

Format: Constructed response, representation analysis, strategy comparison, error analysis, or discourse prompt

Purpose: Check whether students can justify representation matches, analyze function behavior, explain expression structure, and defend the reasonableness of equations as models.

Assessment Ideas

  1. Representation Justification Task Can Be Assessed After Section 1 – Students justify whether multiple representations describe the same relationship using mathematical evidence.
  2. Function Behavior Explanation Task Can Be Assessed After Section 3 – Students compare two function families and explain differences in graph, table, and equation behavior.
  3. Expression Structure Analysis Task Can Be Assessed After Section 4 – Students analyze equivalent expressions and explain why different forms reveal different information.
  4. Equation Model Critique Task Can Be Assessed After Section 5 – Students critique an equation model or solution interpretation and revise the reasoning.
What this tells the teacher:
This level reveals whether students understand functions and algebraic relationships as connected systems of representations rather than isolated procedures. These tasks identify who can explain, justify, critique, and transfer reasoning.

DOK 4 — Rungs 7–8Students can create, compare, revise, and critique models across tables, graphs, equations, contexts, and function families.

Format: Brief performance task, modeling task, critique-and-revise task, design task, or extended application

Purpose: Check whether students can transfer function and algebra foundations to unfamiliar situations, create coherent models, and justify conclusions across multiple representations.

Assessment Ideas

  1. Build a Relationship Web Task Can Be Assessed After Section 1 – Students create a table, graph, equation, and context for the same relationship and justify the connections.
  2. Design a Function Rule Task Can Be Assessed After Section 2 – Students design a function rule for a situation, evaluate it, and explain the meaning of inputs and outputs.
  3. Function Family Classification Challenge Can Be Assessed After Section 3 – Students create examples and non-examples of function families and defend classifications.
  4. Equation as a Model Performance Task Can Be Assessed After Section 5 – Students create, solve, interpret, and critique an equation model for a real-world situation.
What this tells the teacher:
DOK 4 tasks reveal whether students can design, connect, critique, and revise representations and models. These should remain short performance tasks rather than large projects.
