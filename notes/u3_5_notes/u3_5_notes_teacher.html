Intro

Directions

Read the introduction aloud with a partner or in a small group. As you read, annotate the text by highlighting important ideas, circling key vocabulary, and writing short notes about connections, questions, or predictions in the margins.

A system of inequalities has more than one condition. A feasible point is a point that satisfies every inequality in the system. If it fails even one condition, it is not a solution to the system.

Each inequality shades a region of possible points. The overlap of the shaded regions represents the points that satisfy all constraints at the same time. This overlap is called the feasible region.

A candidate point can look reasonable from one boundary but still fail another constraint. Testing every inequality prevents choosing a point that only partly works. This is especially important in contexts with limits like cost, time, or minimum quantities.

Useful representations include systems of inequalities, multiple boundary lines, overlapping shaded regions, feasible points, and constraint-based contexts. Strong work connects the graph to the meaning of the constraints.

Systems of inequalities help model decisions with more than one restriction. A feasible solution represents a possible choice, not necessarily the only possible choice.

Math Notes Feasible Means All Constraints

A feasible point must satisfy every inequality in the system, not just one of them.

$$\begin{aligned}y&\ge x+1\\y&\le -x+7\end{aligned}$$
PointCheck $y\ge x+1$Check $y\le -x+7$Feasible?
$(2,4)$$4\ge 3$ true$4\le 5$ trueYes
$(5,6)$$6\ge 6$ true$6\le 2$ falseNo

The point has to pass both checks to be a solution.

Math Notes Overlapping Solution Regions

A system of inequalities is graphed by showing each inequality and then finding the overlap.

$$\begin{aligned}y&\ge x+1\\y&\le -x+7\end{aligned}$$

Context: A point in the overlap is a choice that meets both constraints at the same time.

Example 2: Graph a System of Inequalities

Graph the system and identify one feasible point.

$$\begin{aligned}y&\ge x-2\\y&\le -x+6\end{aligned}$$
  1. Graph each boundary and shaded region.
  2. Describe the overlap.
  3. Name one feasible point and justify it.

Example 2 answer: Boundaries are solid. The feasible region is above $y=x-2$ and below $y=-x+6$. One feasible point is $(2,2)$ because $2\ge 0$ and $2\le 4$.

Math Notes Testing Candidate Points Efficiently

When only a few points need to be checked, substitution into the inequalities may be faster than drawing a full graph.

$$\begin{aligned}y&\le -x+9\\y&\ge x-1\end{aligned}$$
PointFirst inequalitySecond inequalityDecision
$(3,5)$$5\le 6$ true$5\ge 2$ trueFeasible
$(6,2)$$2\le 3$ true$2\ge 5$ falseNot feasible

Graphing shows the whole feasible region, while point testing quickly checks selected choices.

Example 4: Compare Graphing and Point Testing

Two students analyze the system below.

$$\begin{aligned}y&\le -x+9\\y&\ge x-1\end{aligned}$$

Ava graphs first. Ben tests candidate points first.

  1. Test $(3,5)$ and $(6,2)$ in both inequalities.
  2. Decide which points are feasible.
  3. For only these two points, explain which method is more efficient and why.

Example 4 answer: $(3,5)$ is feasible: $5\le 6$ and $5\ge 2$. $(6,2)$ is not feasible because $2\ge 5$ is false. For two given points, point testing is more efficient than graphing the whole region.
Teacher reminder: Ask students to name the representation they used before they describe the procedure.

Summary

Today you analyzed systems of inequalities by focusing on points that satisfy all constraints.

Each inequality creates a region, and the overlap is the feasible region for the system.

A common error is accepting a point that satisfies only one inequality.

Strong understanding means you can graph the overlap, test candidate points, and interpret feasible choices in context.

Common Mistakes

  • Checking only one inequality. Why it is tempting: one true statement feels like success. What to check instead: test every constraint.
  • Shading union instead of overlap. Why it is tempting: all shaded areas look important. What to check instead: identify where all regions overlap.
  • Mixing dashed and solid boundaries. Why it is tempting: multiple lines increase the load. What to check instead: check each symbol separately.
  • Calling every boundary intersection feasible. Why it is tempting: intersections stand out visually. What to check instead: test the point in all inequalities.
  • Ignoring context restrictions. Why it is tempting: the graph may include negative values. What to check instead: decide whether the situation requires nonnegative or whole-number values.
