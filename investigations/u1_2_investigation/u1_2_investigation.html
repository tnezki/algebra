Algebra 1 - Unit 1

Section 1.2 Investigation

Inputs, Outputs, and Functions

Investigation: The Function Machine Mystery

Time: 15-20 minutes

Format: Function-machine exploration, relationship card sort, matching task, create-a-rule challenge, and class synthesis

Learning Target: Interpret functions as rules that connect inputs and outputs without relying on formal domain and range language.
Investigation Directions

Setup

  • Grouping: Partners or groups of 3.
  • Materials: Investigation handout, pencil, and scissors only if using physical cut cards.
  • Teacher setup: Project a simple function machine: Input → add 4 → Output. Ask, “What would come out if I put in 6? What about 10? How do you know?”
  • Timing: Launch 2-3 min, Part 1 4 min, Part 2 5 min, Part 3 4 min, Part 4 3 min, synthesis 3-4 min.

Student Goal

Today you will investigate how an input is changed into an output.

Input → Rule → Output

By the end, you should be able to explain what makes a rule a function and why one input cannot have two different outputs in the same function.

Norms

  • Predict before calculating.
  • Test more than one input before deciding on a rule.
  • Look for what stays consistent.
  • Use evidence from the table, context, mapping, or rule.

Class Notes / Final Discovery
Teaching Resource 1: Teacher Script

Purpose

This investigation helps students discover the function idea before formal vocabulary becomes the focus. Students should leave with the idea that a function is a rule that assigns exactly one output to each input. Domain and range language is intentionally avoided.

Launch: 2-3 minutes

Say: “Today we are investigating machines. Each machine takes something in, follows a rule, and sends something out. Your job is to decide what makes a machine reliable enough to be called a function.”

Display: Input → add 4 → Output.

Ask: “If I put in 6, what comes out? If I put in 6 again, should I get the same output or can it change?”

Teacher Moves

  1. Part 1: Ask students to verbalize the rule rather than only calculate.
  2. Part 2: Let students sort before defining function. When groups disagree, ask, “Which input should we check?”
  3. Part 3: Push students to identify input and output in the context, not just in the table.
  4. Part 4: Check that student rules do not assign two outputs to the same input.
Teaching Resource 2: Observations and Misconceptions

Expected Student Observations

  • A machine can be written as a table, equation, context, or mapping.
  • The input is the value chosen or put into the rule.
  • The output is the result after the rule is applied.
  • Different inputs can share an output. For example, $-2$ and 2 both give 4 in $y=x^2$.
  • The problem happens when the same input has two different outputs.

Common Misconceptions

  • “Outputs have to be different.” Point to Card C or Card G and ask which input repeats.
  • “A function has to be an equation.” Ask students to identify function behavior in a context or mapping.
  • “If a number appears twice anywhere, it is not a function.” Ask whether the repeated number is an input or an output.
  • Students reverse input and output. Re-read the context and label what is chosen first.

Closure

Final consensus statement: A function is a rule that assigns exactly one output to each input.

Teaching Resource 4: Answer Key Continued
Part 3: Match the Rule

Original task: Explain how the rewards program context, machine, table, and equation show the same function.

Answer: The input is levels completed. The output is total points. The 10 is the starting amount, the table output when $x=0$, and the constant in $y=2x+10$. The 2 is the points earned for each level, the repeated table increase, and the coefficient of $x$.

Part 4: Create a Machine

Sample answer: Rule: multiply by 3, then subtract 1. Table: 0 → -1, 1 → 2, 2 → 5, 5 → 14. This is a function because each input has exactly one output.

Part 5 and Exit Check
  1. An input is the value chosen or put into a rule.
  2. An output is the result after the rule is applied.
  3. A rule tells how to change the input into the output.
  4. A function must give each input exactly one output.
  5. Not a function: the same input has more than one output.

Exit Check: Function; Not a function; Function.
